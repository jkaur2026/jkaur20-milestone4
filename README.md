# jkaur20-milestone4

This initiative establishes a distributed feature engineering pipeline utilizing Apache Spark. café sales data is created and analyzed through distributed transformations to calculate aggregated metrics like beverage popularity and total café sales.

This project aims to assess the scalability of feature engineering tasks in distributed data processing frameworks while ensuring reproducibility.

The pipeline illustrates how Spark allocates transformations and aggregations over partitions to effectively handle large datasets

The structure of the repository:
pipeline.py whcih Distributed Spark feature engineering pipeline
generate_data.py which is the script used to generate synthetic café sales data
requirements.txt which is the Python dependencies required for the project
test_data/ which is the small dataset used for pipeline testing
results/  which is the output generated by the Spark pipeline
README.md which is the instructions of the whole setup 
REPORT.md which is the  Performance analysis and system discussion

SETUP
Clone the repository: https://github.com/jkaur2026/jkaur20-milestone4.git
Create a Python virtual environment:python3 -m venv .venv
Activate the virtual environment
source .venv/bin/activate
install the dependencies:pip install -r requirements.txt
The dataset is generated using a seeded random generator to ensure reproducibility so we did the following below:
Generate a small dataset used for testing:
python generate_data.py --rows 1000 --seed 42 --output test_data/
Generate the large dataset:
python generate_data.py --rows 10000000 --seed 42 --output large_data/
Run the pipeline on the small dataset to verify correctness:
python pipeline.py --input test_data --output results
Run the pipeline on the large  dataset:
python pipeline.py --input large_data --output results_large

The Spark pipeline generates two main output directories.
results/drink_popularity/
results/cafe_sales/

Execution time was measured using the time command.
Small dataset runtime:
real 0m25.220s
Large dataset runtime:
real 1m32.359s
Additional performance analysis and discussion are provided in REPORT.md.

This thing guarantees consistent experimentation by:
• random seed implemented in generate_data.py
• predictable transformations in the Spark pipeline
• clear dependency setup via requirements.txt
Executing the data generation script with the same seed yields identical datasets
